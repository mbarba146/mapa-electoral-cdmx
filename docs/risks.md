# Risks

## Riesgo 1: datos oficiales en formatos distintos

Mitigación: usar un esquema intermedio normalizado.

## Riesgo 2: alcance demasiado grande

Mitigación: priorizar dashboard, comparador, documentación y datos mock.

## Riesgo 3: complejidad del mapa

Mitigación: iniciar con mapa conceptual o grid de alcaldías.

## Riesgo 4: backend y base de datos pueden consumir mucho tiempo

Mitigación: comenzar con CSV/JSON y dejar PostgreSQL como mejora posterior.

## Riesgo 5: noticias pueden confundirse con causalidad electoral

Mitigación: separar noticias como contexto y aclararlo en metodología.
