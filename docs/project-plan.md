# Project Plan

## Proyecto

Mapa Electoral CDMX

## Objetivo

Construir una aplicación web analítica para explorar patrones históricos de votación en la Ciudad de México por alcaldía.

## Entrega

Martes.

## Enfoque

El proyecto prioriza aprendizaje acelerado, gestión del alcance, documentación y construcción incremental.

## Fases

### Jueves
- Crear cuenta de GitHub.
- Crear repositorio.
- Definir estructura inicial.
- Documentar alcance.
- Crear prototipo visual.

### Viernes
- Crear frontend base.
- Crear navegación.
- Crear páginas principales.
- Cargar datos mock.

### Sábado
- Construir dashboard electoral.
- Crear comparador.
- Crear descarga CSV.

### Domingo
- Crear backend con FastAPI.
- Dockerizar servicios.
- Documentar ejecución local.

### Lunes
- Agregar noticias políticas.
- Completar metodología.
- Pulir UI.

### Martes
- QA.
- Revisión final.
- Entrega.
